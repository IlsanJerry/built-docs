Search experiencesedit

Enterprise Search provides different tools to create search experiences for both App Search and Workplace Search.

  • Generated and custom search experiences for App Search
  • Default search experience for Workplace Search
  • Custom search experiences for Workplace Search

Generated and custom search experiences for App Searchedit

Depending on customization needs, App Search provides the following options for creating search experiences:

  • You can use Search UI to:

    • Generate a search experience preview using App Search.
    • Download and customize this search experience for embedding into your own deployment.
  • Use Search UI React open source library to create a custom search experience and embed it into a web application.
  • Search API provides a REST API to use App Search capabilities from any application.
  • API clients provide libraries for communicating with App Search APIs using different programming languages. API clients makes easier for developers to create integrations with App Search.

Default search experience for Workplace Searchedit

Workplace Search provides a powerful and modern web-based search experience. See Searcher’s Guide to Workplace Search for a summary of all its capabilities.

You can use private sources in the private dashboard area. This allows you to customize your searchable content.

Custom search experiences for Workplace Searchedit

You can also integrate Workplace Search into existing applications. This way, users will be able to access search results using their current tools, so there is no need for separate searching experiences.

See Building Custom Search Experiences to connect an existing enterprise application with Workplace Search.

You can use Search API and API clients to build custom search experiences on top of Workspace Search capabilities.

Management experiencesedit

You can manage Enterprise Search using the management interfaces. There are different interfaces that you can choose for Enterprise Search management depending on your preferences and the features of Enterprise Search to manage.

Learn more about the management interfaces:

  • Standalone Enterprise Search
  • Enterprise Search in Kibana (beta)
  • Additional Elastic interfaces
  • Choose a management experience for Enterprise Search

Standalone Enterprise Searchedit

Standalone Enterprise Search is the web based interface used to manage Enterprise Search as a standalone product. It focuses on Enterprise Search and does not manage other aspects of the Elastic Stack.

A single log-in is used to access Enterprise Search. See Logging in to Enterprise Search for login details, and Users and Access for details on configuring user access.

Once logged in, users will be able to access one or both products within Enterprise Search, depending on their permissions:

  • Elastic App Search
  • Elastic Workspace Search

The standalone interface displays a navigation bar on the left, providing access to:

  • Product specific features
  • User Management
  • Enterprise Search Home
  • Settings
  • Logout links
  • User profile information

Standalone interface is the default management interface for Enterprise Search 7.x. It will remain available through 7.x versions.

This interface will be removed in 8.0.0 version. You should start getting familiar with Kibana management interface before 8.0 version.

Enterprise Search in Kibana (beta)edit

Enterprise Search Kibana management interface is included in Kibana. You can now manage Enterprise Search features and the rest of the Elastic stack from a single interface.

Kibana management interface can be used in addition to, or in place of, Standalone Enterprise Search.

Access Enterprise Search from the Kibana main menu. See the Where to start in the Kibana documentation for details on navigating Kibana.

After navigating to Enterprise Search, choose Elastic App Search or Elastic Workspace Search:

After selecting a product, use the navigation within Kibana to access the following:

  • Product specific features
  • User Management
  • Enterprise Search Home
  • Settings
  • Logout links
  • User profile information

The Kibana menu can be accessed any time clicking on the menu icon in order to navigate to a different Elastic solution.

The Kibana management interface is a beta feature. Beta features are subject to change and are not covered by the support SLA of general release (GA) features. Elastic plans to promote this feature to GA in 8.0 release.

As of 8.0 release, this interface will be the only one available. So start using it from today to start the transition to Kibana as a single entry point for the Elastic Stack.

Additional Elastic interfacesedit

Some features relevant to Enterprise Search are managed outside of Enterprise Search management interfaces. They are accessible from within your Elastic deployment.

Some examples include:

  • Web crawler events logs. See Web crawler events logs
  • Access to Workplace Search source activity logs. See troubleshooting common source sync issues.
  • Workplace Search analytics events. See Workplace Search analytics.
  • Index Lifecycle Management for Log retention.
  • Snapshots management for Migrating from App Search on Elastic Cloud and Migrating from self-managed App Search

Choose a management experience for Enterprise Searchedit

You can use both management interfaces, but you should take the following points into account for choosing:

Authentication

You will need to log into each of the management interfaces separately.

OAuth redirect URIs are different for each interface. You will need to setup your OAuth identity provider for the different interfaces.

Interface

Enterprise Search in Kibana provides the following benefits over Standalone Enterprise Search:

  • Dark mode, selectable in Kibana settings
  • Mobile and tablet friendly design.
  • Screen reader accessibility.
  • Chinese and Japanese localization.
  • More consistent with the rest of the Elastic product stack.

Features

Standalone Enterprise Search and Enterprise in Kibana are almost functionally equivalent. The only visible change is that Meta Engines are merged with Engines in the Kibana interface.

Some features are only available in one of the interfaces. You will need to switch to the appropriate interface to manage the following features:

  • Standalone interface: Web Crawler.
  • Kibana interface: Precision Tuning UI.

Troubleshoot Enterprise Search in Kibanaedit

From https://github.com/elastic/app-search-team/issues/1848: Enterprise Search in Kibana is disabled when:

  • Feature control through Kibana spaces
  • With RBAC disabled in Enterprise Search
  • With RBAC enabled in Enterprise Search, but no access
  • With access to App Search
  • With access to Workplace Search
  • With access but when Configuration is not set
  • With access but when HTTP call exceeds the timeout
  • With access but when Enterprise Search is unavailable

From Constance: The setup guide (see screenshot above) also lists several other things that can go wrong. For clarity, I’ll list them here:

  • Kibana and Enterprise Search not pointed to the same Elasticsearch instance/clusters
  • Kibana and Enterprise Search not on the same authentication methods (e.g. different SAML providers)
  • Firewall or some other DNS/proxy issue preventing Kibana from reaching Enterprise Search
  • Enterprise Search on standard auth (not sure this applies anymore with the 7.14 user changes)
  • Logged into Kibana as a user who does not have access to Enterprise Search

Search Relevance Suggestions APIedit

Search Relevance Suggestions are only available for Platinum Licenses.

Search Relevance Suggestions are automated suggestions for relevance improvement, and are automatically maintained for each engine. Use this API to retrieve and manage Search Relevance Suggestions.

  • List Search Relevance Suggestions for an Engine
  • List Search Relevance Suggestions for an Engine Query

List Search Relevance Suggestions for an Engineedit

Return a list of Search Relevance Suggestions for an engine.

GET /api/as/v1/engines/{ENGINE_NAME}/search_relevance_suggestions
POST /api/as/v1/engines/{ENGINE_NAME}/search_relevance_suggestions

Request body

filters (optional)

Additional filters for retrieving Search Relevance Suggestions:

  • type: Search Relevance Suggestion type to retrieve. Only curation type is available for now.
  • status: Array of statuses for the Search Relevance Suggestions. Must be one of pending, applied,rejected,disabled.

Both filters are optional. In case both are specified, results will comply with both type and status requested (AND of both filters).

Usage examples

Request all Search Relevance Suggestions for an engine:

curl -X GET 'https://[instance id].ent-search.[region].[provider].cloud.es.io/api/as/v1/engines/national-parks-demo/search_relevance_suggestions' \
-H 'Authorization: Bearer search-soaewu2ye6uc45dr8mcd54v8' \

Request Search Relevance Suggestions for an engine, filtering by status:

curl -X POST 'https://[instance id].ent-search.[region].[provider].cloud.es.io/api/as/v1/engines/national-parks-demo/search_relevance_suggestions' \
-H 'Content-Type: application/json' \
-H 'Authorization: Bearer search-soaewu2ye6uc45dr8mcd54v8' \
-d '{
  "filters": {
    "status": ["pending", "applied"]
  }
}'

Request Search Relevance Suggestions for an engine, filtering by Search Relevance Suggestion type:

curl -X POST 'https://[instance id].ent-search.[region].[provider].cloud.es.io/api/as/v1/engines/national-parks-demo/search_relevance_suggestions' \
-H 'Content-Type: application/json' \
-H 'Authorization: Bearer search-soaewu2ye6uc45dr8mcd54v8' \
-d '{
  "filters": {
    "type": "curation"
  }
}'

The following fields are returned for each result:

  • query: Query that the Search Relevance refers to
  • type: Search Relevance Suggestion type (curation for now)
  • status: One of pending, applied, automated, rejected, disabled.
  • updated_at: Timestamp of the last update made to the suggestion.
  • created_at: Timestamp of the suggestion creation date and time.
  • promoted: Array of document identifiers that are being promoted by the suggestion.
  • operation: Identifies the operation to be done to the query curation. One of create, update or delete.
  • curation_id: (Optional) Curation identifier that the operation will be applied to.
  • override_curation_id: (Optional) Curation identifier for an status pending ,create operation that identifies the curation that will be overriden with the suggestion.

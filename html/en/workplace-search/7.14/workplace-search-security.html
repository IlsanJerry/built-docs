Manage users and access to Workplace Searchedit

You manage users' access to Workplace Search using Workplace Search mappings for roles and groups. Workplace search provides default mappings. And you can define your own custom mappings.

You can manage users for your Elastic deployment from various interfaces within or external to your deployment. You can manage users from within Workplace Search.

Default Workplace Search mappingsedit

These map users, based on their Elastic Stack roles, to Workplace Search roles and group access.

All users with the given Elastic Stack role match the mapping and are assigned to the Workplace Search role and groups specified in the mapping.

Some of these mappings are always active, while others are active only when Enterprise Search role-based access control (RBAC) is enabled or disabled. See Enterprise Search doc.

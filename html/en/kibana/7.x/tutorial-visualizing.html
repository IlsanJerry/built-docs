Visualize your dataedit

Shape your data using a variety of Kibana supported visualizations, tables, and more. In this tutorial, you’ll create four visualizations that you’ll use to create a dashboard.

To begin, open the menu, go to Dashboard, then click Create new dashboard.

Compare the number of speaking parts in the playsedit

To visualize the Shakespeare data and compare the number of speaking parts in the plays, create a bar chart using Lens.

  1. Click Create new, then click Lens on the New Visualization window.

  2. Make sure the index pattern is shakes.

  3. Display the play data along the x-axis.

    1. From the Available fields list, drag and drop play_name to the X-axis field.
    2. Click Top values of play_name.
    3. From the Order direction dropdown, select Ascending.
    4. In the Label field, enter Play Name.

  4. Display the number of speaking parts per play along the y-axis.

    1. From the Available fields list, drag and drop speaker to the Y-axis field.
    2. Click Unique count of speaker.

    3. In the Label field, enter Speaking Parts.

  5. Save the chart with the name Bar Example.

    To show a tooltip with the number of speaking parts for that play, hover over a bar.

    Notice how the individual play names show up as whole phrases, instead of broken up into individual words. This is the result of the mapping you did at the beginning of the tutorial, when you marked the play_name field as not analyzed.

View the average account balance by ageedit

To gain insight into the account balances in the bank account data, create a pie chart. In this tutorial, you’ll use the Elasticsearch bucket aggregation to specify the pie slices to display. The bucket aggregation sorts the documents that match your search criteria into different categories and establishes multiple ranges of account balances so that you can find how many accounts fall into each range.

  1. Click Create new, then click Pie on the New Visualization window.

  2. On the Choose a source window, select ba*.

    Since the default search matches all documents, the pie contains a single slice.

  3. In the Buckets pane, click Add > Split slices.

    1. From the Aggregation dropdown, select Range.
    2. From the Field dropdown, select balance.

    3. Click Add range until there are six rows of fields, then define the following ranges:

      0             999
1000         2999
3000         6999
7000        14999
15000       30999
31000       50000

  4. Click Update.

    The pie chart displays the proportion of the 1,000 accounts that fall into each of the ranges.

  5. Add another bucket aggregation that displays the ages of the account holders.

    1. In the Buckets pane, click Add, then click Split slices.
    2. From the Sub aggregation dropdown, select Terms.

    3. From the Field dropdown, select age, then click Update.

      The break down of the ages of the account holders are displayed in a ring around the balance ranges.

  6. Click Save, then enter Pie Example in the Title field.

Visualize geographic informationedit

To visualize geographic information in the log file data, use Maps.

  1. Click Create new, then click Maps on the New Visualization window.

  2. To change the time, use the time filter.

    1. Set the Start date to May 18, 2015 @ 12:00:00.000.

    2. Set the End date to May 20, 2015 @ 12:00:00.000.

    3. Click Update

  3. Map the geo coordinates from the log files.

    1. Click Add layer > Clusters and grids.
    2. From the Index pattern dropdown, select logstash.
    3. Click Add layer.

  4. Specify the Layer Style.

    1. From the Fill color dropdown, select the yellow to red color ramp.
    2. In the Border width field, enter 3.

    3. From the Border color dropdown, select #FFF, then click Save & close.

  5. Click Save, then enter Map Example in the Title field.

  6. Add the map to your dashboard.

    1. Open the menu, go to Dashboard, then click Add.
    2. On the Add panels flyout, click Map Example.

Add context to your visualizations with Markdownedit

Add context to your new visualizations with Markdown text.

  1. Click Create new, then click Markdown on the New Visualization window.

  2. In the Markdown text field, enter:

    # This is a tutorial dashboard!
The Markdown widget uses **markdown** syntax.
> Blockquotes in Markdown use the > character.

  3. Click Update.

    The Markdown renders in the preview pane.

  4. Click Save, then enter Markdown Example in the Title field.

Next stepsedit

Now that you have the basics, you’re ready to start exploring your own system data with Kibana.

  • To add your own data to Kibana, refer to Add data to Kibana.
  • To search and filter your data, refer to Discover.
  • To create a dashboard with your own data, refer to Dashboard.
  • To create maps that you can add to your dashboards, refer to Maps.
  • To create presentations of your live data, refer to Canvas.

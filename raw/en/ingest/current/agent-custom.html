2. Elastic Agent to Elasticsearch using a custom integrationedit

You can host Elasticsearch on your own hardware or send your data to Elasticsearch on Elastic Cloud. For most users, Elastic Agent writing directly to Elasticsearch on Elastic Cloud provides the easiest and fastest time to value. Our hosted Elasticsearch Service is available on AWS, GCP, and Azure, and you can try it for free.

Ingest models
  • Data from syslog server: TCP logs (custom) or UDP logs (custom) to Elasticsearch
  • Data from scraping an external API: API (httpjson) to Elasticsearch
  • Data pushed from external API to HTTP endpoint: Http Endpoint logs (custom) to Elasticsearch
  • Data read from disk: Custom logs [category]: Logs (custom) to Elasticsearch
Use when
No dedicated Elastic Agent integration exists, but a customizable integration is available

Process overviewedit

  • Find the integration for your custom data source. In Kibana, go to Management> Integrations.
  • Enable the integration and set up Elastic Agent.
  • Launch discover/dashboard to explore.

For details and next steps, check out the Elastic Agent integrations docs for your data source.

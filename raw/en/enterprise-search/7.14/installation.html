Installationedit

Deploy Elastic Enterprise Search on Elastic Cloud or run the solution on-premises.

Get started now with a free Elastic Cloud trial:

  • Deploying on Elastic Cloud

Or, run self-managed deployments on your own infrastructure:

  • Running on ECK (Elastic Cloud on Kubernetes)
  • Running using Linux & MacOS packages
  • Running using Docker images

Deploying on Elastic Cloudedit

Deploy Enterprise Search on Elastic Cloud with the following steps. For more details, see the relevant Elastic Cloud documentation.

Step 1. Log in to Elastic Cloud. If you’re using Elastic Cloud for the first time, create an account. New accounts include a free trial.

Step 2. Within Elastic Cloud, navigate to Create deployment to create your Enterprise Search deployment. If you’re using a new account, your free trial starts you on this screen automatically.

Choose Elastic Enterprise Search.

Optionally, change the values of other fields, such as cloud provider and geographic region.

Click Create deployment to deploy Enterprise Search.

Step 3. While waiting for the deployment to start, download or copy the elastic user password shown on the screen. You don’t need these credentials now, but they are shown only once. Record them securely.

When available, choose Open Enterprise Search, which redirects you to the Enterprise Search home screen.

You’re ready to use Enterprise Search!

From the Enterprise Search home screen, select the product for your use case. Need help choosing the best product for your needs? Learn more:

  • Elastic App Search
  • Elastic Workplace Search

Then continue with the Getting Started documentation for your selected product:

  • Getting started with App Search
  • Getting started with Workplace Search

Resetting the Enterprise Search passwordedit

If you lose the password for the elastic user, you will need to reset it through the Elastic Cloud console. Navigate to the specific deployment within Elastic Cloud, and then click Manage and Reset password to get to the Security screen. From there, click Reset password to complete the process.

Running using Linux/MacOS packagesedit

Elastic distributes Linux and MacOS packages for installing Elasticsearch, Kibana and Enterprise Search within your development environment or on production systems you have provisioned.

Before installing Enterprise Search packages, you need to satisfy all dependencies:

  • 64-bit Linux or MacOS (10.13+).
  • Java 8 or Java 11.
  • 6GB of available RAM (4GB if running Elasticsearch on a separate machine).
  • 5GB of available storage.

Next, you will need to get Elasticsearch up and running with security features enabled:

Step 1. Download Elasticsearch 7.14.0 and navigate to the newly unpacked directory.

Step 2. Locate the config/elasticsearch.yml configuration file and edit the following setting to enable Elasticsearch’s Security features:

xpack.security.enabled: true
xpack.security.authc.api_key.enabled: true

Step 3. Start Elasticsearch:

For development environments, you can simply run the following command:

$ bin/elasticsearch

In production environments, you need to ensure the service will restart after a server reboot. Please refer to your operation system documentation for more details on how to achieve that (official RPM and Debian packages enable the service automatically via systemd).

Step 4. Open a new terminal window, and navigate to the Elasticsearch installation directory. From here, we must generate a random password for the elastic user, which will be used by Enterprise Search to securely communicate with the underlying Elasticsearch cluster:

$ bin/elasticsearch-setup-passwords auto

Locate the passwords for elastic and kibana_system users in the console output and keep those handy: we will need them in a few moments.

You may also manually generate passwords using the elasticsearch-setup-passwords command.

Next, can configure and start Kibana or move on to Enterprise Search configuration.

It is not required for Enterprise Search 7.14.0, but really soon Kibana will become the main interface for working with the solution and you may want to start using it now.

If you just want to explore the Enterprise Search interface, feel free to jump ahead to Step 8.

Step 5. Download Kibana 7.14.0 and navigate to the newly unpacked directory.

Step 6. Locate config/kibana.yml and add the following configuration:

elasticsearch.username: kibana_system
elasticsearch.password: [KIBANA_SYSTEM_USER_PASSWORD]
enterpriseSearch.host: http://localhost:3002

Replace KIBANA_SYSTEM_USER_PASSWORD with the kibana_system user password generated in step 4.

Step 7. Start Kibana

For development environments, you can simply run the following command:

$ bin/kibana

In production environments, you need to ensure the service will restart automatically after a server reboot.

We must now configure Enterprise Search:

Step 8. Download Enterprise Search 7.14.0 and navigate to the newly unpacked directory.

Step 9. Locate the config/enterprise-search.yml file and edit the following settings using the recently generated password:

elasticsearch.username: elastic
elasticsearch.password: [ELASTIC_USER_PASSWORD]

Step 10. Add the following configuration to the config/enterprise-search.yml file. It allows Enterprise Search to create to appropriate indices for the target Elasticsearch cluster:

allow_es_settings_modification: true

allow_es_settings_modification: true can be used safely if the cluster is used for Enterprise Search only. An alternative option is to set auto_create_index at the cluster level. See Elasticsearch cluster settings for more information.

Step 11. You must also set at least one encryption key in the same config/enterprise-search.yml configuration file (256-bit key recommended), as an array:

secret_management.encryption_keys: [{encryption_key1}, {encryption_key2}, {...}]

For more information on encryption keys for Enterprise Search, refer to the Configure encryption keys guide.

Step 12. Run the boot command and provide a default user password to start Enterprise Search. . For those using Workplace Search, the default password will also be used to log in to the Workplace Search administrative experience:

$ ENT_SEARCH_DEFAULT_PASSWORD=passwordexample bin/enterprise-search

Should you choose not to provide a default password, a random passphrase will be generated for you on the first boot sequence, and will be output in the console

Step 13. Now you can choose which UI you are going to use for accessing Enterprise Search. No matter which option you choose, use the enterprise_search user and the password provided at boot time to log in.

  • If you have installed, configured and started Kibana (steps 5-7 above), you can head over to http://localhost:5601 to access Enterprise Search via Kibana. After logging in, you will end up on the Kibana Home screen, where you can click on the "Enterprise Search" tile to access the solution.
  • Otherwise, you can access the Enterprise Search UI at http://localhost:3002. You will be presented with a one-time deployment status report. If the Elasticsearch cluster is not already running with a Platinum license applied, a trial license is automatically applied for you and will be valid for 30 days.

Elasticsearch cluster settingsedit

Enterprise Search makes alterations to the auto_create_index persistent and transient settings within Elasticsearch when using allow_es_settings_modification: true. Avoid making changes to those settings as it may result in unintended consequences.

An alternative approach involves setting the auto_create_index configuration in elasticsearch.yml for the target Elasticsearch cluster:

action.auto_create_index: ".ent-search-*-logs-*,-.ent-search-*,-test-.ent-search-*,+*"

Using this alternative approach allows you to maintain all Elasticsearch-specific settings at the cluster level.

Resetting the Enterprise Search passwordedit

Run the binary with an attached --reset-auth command:

  • Standard Auth: Resets login and generates new random password.
  • elasticsearch-native and elasticsearch-saml: Removes the current role mappings. All users are granted the owner role.

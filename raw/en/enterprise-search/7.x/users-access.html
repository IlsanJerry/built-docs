Enterprise Search role-based access control (RBAC)edit

Access to App Search and Workplace Search is affected by an Enterprise Search setting called role-based access control (RBAC).

Solution-wide setting. You enable/disable from within a particular product. But it applies to both products.

Disabled by default for a new deployment. Enabled by default for an upgraded deployment.

  • Enable Enterprise Search RBAC
  • Disable Enterprise Search RBAC

Enabling it changes the default mappings for each product. Changes mappings from permissive to restrictive. See the following sections in the product-specific documentation:

  • Default App Search mappings
  • Default Workplace Search mappings

Must enable it to use custom mappings for each product. See the following sections in the product-specific documentation:

  • Default App Search mappings
  • Default Workplace Search mappings

Manage users for your Elastic deploymentedit

A user/account is composed of an identifier (username) and potentially other attributes (like email).

Within a variety of interfaces you can list, create, invite, and delete users, and modify their attributes. Specific functionality varies by interface. Manage users for your Elastic deployment with any of the following interfaces:

  • Manage users using App Search
  • Manage users using Workplace Search
  • Manage users using the Elastic Stack
  • Manage users using external identity provider services (IdPs)
  • Manage users using Elastic Cloud

Manage users using App Searchedit

Can create, invite, and list users. Directly within App Search.

You create these users and give them access to App Search at the same time. See Manage users within App Search in the App Search documentation.

Must enable Enterprise Search role-based access control (RBAC), which also affects the default mappings for App Search and Workplace Search. See Enterprise Search role-based access control (RBAC).

These are native Elastic users. See Native user authentication in the Elasticsearch documentation.

Manage users using Workplace Searchedit

Can create, invite, and list users. Directly within Workplace Search.

You create these users and give them access to App Search at the same time. See Manage users within Wrokplace Search in the Workplace Search documentation.

Must enable Enterprise Search role-based access control (RBAC), which also affects the default mappings for App Search and Workplace Search. See Enterprise Search role-based access control (RBAC).

These are native Elastic users. See Native user authentication in the Elasticsearch documentation.

Manage users using the Elastic Stackedit

Elasticsearch and Kibana provide UI, API, CLI, and file interfaces to list, create, delete, and modify users for your deployment.

Manage users who have broad access to your Elastic deployment. App Search, Workplace Search, other Elastic solutions, apps, and features.

To manage these users, see the following information in the Elasticsearch documentation:

  • Native user authentication
  • File-based user authentication

To give these users access to App Search and Enterprise Search, you must map their Elastic Stack roles or other user attributes to App Search or Workplace Search roles.

See the following guides in the App Search and Workplace Search documentation:

  • Manage users and access to App Search
  • Manage users and access to Workplace Search

Manage users using external identity provider services (IdPs)edit

SAML 2.0 Identity Provider service (IdP). Examples: Okta, Auth0, OneLogin. Specific interfaces vary by service provider.

Manage users that you can use across Elastic deployments and other web applications.

See SAML authentication in the Elasticsearch documentation.

You must complete a one-time setup process for each SAML identify provider you’d like to use with your deployment. See Set up a SAML identity provider (IdP).

After completing the one-time setup, manage these users within the interfaces provided by the IdP.

To give these users access to App Search and Enterprise Search, you must map their Elastic Stack roles or other user attributes to App Search or Workplace Search roles.

See the following guides in the App Search and Workplace Search documentation:

  • Manage users and access to App Search
  • Manage users and access to Workplace Search

Manage users using Elastic Cloudedit

Elastic Cloud is a SAML 2.0 IdP managed by Elastic. However, presently, you can have only one Elastic Cloud user per deployment.

See Elastic Cloud user.

That user creates the deployment. That user is a superuser for the deployment.

Default usersedit

After starting Enterprise Search, the following users already exist and can access Enterprise Search.

Several default users who can access Enterprise Search.

  • Elastic Cloud user
  • elastic user
  • enterprise_search user

Elastic Cloud useredit

Provisioned when a person creates a new Elastic Cloud account. It’s the user that creates the Elastic Cloud deployment.

Co-default superuser for the deployment (with elastic). elastic user.

elastic useredit

Provisioned when the deployment is created (Cloud) or during the process to enable security (self-managed).

One of the built-in users for the Elastic Stack. See Built-in users in the Elasticsearch documentation.

Co-default superuser for the deployment (with Elastic Cloud user). Elastic Cloud user.

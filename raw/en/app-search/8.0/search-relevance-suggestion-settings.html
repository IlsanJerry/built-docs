Search Relevance Suggestions Settings APIedit

Search Relevance Suggestions Settings are only available for Platinum Licenses.

Search Relevance Suggestions Settings are a set of engine-level configurations to determine how Search Relevance Suggestions are used. Use this API to retrieve and manage Search Relevance Suggestions Settings.

  • Get Search Relevance Suggestions Settings for an Engine
  • Update Search Relevance Suggestions Settings for an Engine

Get Search Relevance Suggestions Settings for an Engineedit

Return the Search Relevance Suggestions Settings for an engine.

GET /api/as/v1/engines/{ENGINE_NAME}/search_relevance_suggestions/settings

Usage examples

Retrieve Search Relevance Suggestions Settings for an engine:

curl -X GET 'https://[instance id].ent-search.[region].[provider].cloud.es.io/api/as/v1/engines/national-parks-demo/search_relevance_suggestions/settings' \
-H 'Authorization: Bearer private-xxxxxxxxxxxxxxxxxxxxxxxx'

Response:

{
  "curation": {
    "enabled": true,
    "mode": "manual",
    "timeframe": 7,
    "max_size": 3,
    "min_doc_position": 3,
    "min_clicks": 20,
    "schedule_frequency": 1,
    "schedule_unit": "day"
  }
}

Update Search Relevance Suggestions Settingsedit

Update the Search Relevance Suggestions Settings for an engine.

PUT /api/as/v1/engines/{ENGINE_NAME}/search_relevance_suggestions/settings

Request body

curations

The Search Relevance Suggestions Settings for the curation type:

  • enabled: Boolean to determine if the Search Relevance Suggestions is turned on for the engine and curation type.
  • mode: Must be manual or automatic. Curations are automatically applied if automatic. manual by default.
  • timeframe: Integer to determine the number of days worth of logs to use for the Search Relevance Suggestions analysis. Defaults to 7.
  • max_size: Maximum number of promoted documents. Defaults to 3.
  • min_doc_position: consider only curation with at least a doc at the position. Defaults to 3.
  • min_clicks: Minimum number of clicks to be considered as significant. Defaults to 20.
  • schedule_frequency: Integer value used with schedule_unit to determine how frequently to run Search Relevance Suggestions analysis. Defaults to 1.
  • schedule_unit: Value used with schedule_frequency to determine how frequently to run Search Relevance Suggestions analysis. Must be one of second, minute, hour, day, week, month. Defaults to day.

All fields are optional.

Usage examples

Turn off Search Relevance Suggestions for an engine:

curl -X PUT 'https://[instance id].ent-search.[region].[provider].cloud.es.io/api/as/v1/engines/national-parks-demo/search_relevance_suggestions/settings' \
-H 'Authorization: Bearer private-xxxxxxxxxxxxxxxxxxxxxxxx' \
-d '{ "enabled": false }'

Response:

{
  "curation": {
    "enabled": false,
    "mode": "manual",
    "timeframe": 7,
    "max_size": 3,
    "min_doc_position": 3,
    "min_clicks": 20,
    "schedule_frequency": 1,
    "schedule_unit": "day"
  }
}

Turn on automatic Search Relevance Suggestions for an engine:

curl -X PUT 'https://[instance id].ent-search.[region].[provider].cloud.es.io/api/as/v1/engines/national-parks-demo/search_relevance_suggestions/settings' \
-H 'Authorization: Bearer private-xxxxxxxxxxxxxxxxxxxxxxxx' \
-d '{ "mode": "automatic" }'

Response:

{
  "curation": {
    "enabled": false,
    "mode": "automatic",
    "timeframe": 7,
    "max_size": 3,
    "min_doc_position": 3,
    "min_clicks": 20,
    "schedule_frequency": 1,
    "schedule_unit": "day"
  }
}

Adaptive relevance API referenceedit

Adaptive Relevance Suggestions APIedit

Adaptive Relevance Suggestions are only available for Platinum Licenses.

Adaptive Relevance Suggestions are automated suggestions for relevance improvement, and are automatically maintained for each engine. Use this API to retrieve and manage Adaptive Relevance Suggestions.

  • List Adaptive Relevance Suggestions for an Engine
  • List Adaptive Relevance Suggestions for an Engine Query

List Adaptive Relevance Suggestions for an Engineedit

Return a list of Adaptive Relevance Suggestions for an engine.

GET /api/as/v1/engines/{ENGINE_NAME}/adaptive_relevance/suggestions
POST /api/as/v1/engines/{ENGINE_NAME}/adaptive_relevance/suggestions

Request body

filters (optional)

Additional filters for retrieving Adaptive Relevance Suggestions:

  • type: Adaptive Relevance Suggestion type to retrieve. Only curation type is available for now.
  • status: Array of statuses for the Adaptive Relevance Suggestions. Must be one of pending, applied,rejected,disabled.

Both filters are optional. In case both are specified, results will comply with both type and status requested (AND of both filters).

Request Adaptive Relevance Suggestions for an engine, filtering by status:

curl -X POST 'https://[instance id].ent-search.[region].[provider].cloud.es.io/api/as/v1/engines/national-parks-demo/adaptive_relevance/suggestions' \
-H 'Content-Type: application/json' \
-H 'Authorization: Bearer search-soaewu2ye6uc45dr8mcd54v8' \
-d '{
  "filters": {
    "status": ["pending", "applied"]
  }
}'

Request Adaptive Relevance Suggestions for an engine, filtering by Adaptive Relevance Suggestion type:

curl -X POST 'https://[instance id].ent-search.[region].[provider].cloud.es.io/api/as/v1/engines/national-parks-demo/adaptive_relevance/suggestions' \
-H 'Content-Type: application/json' \
-H 'Authorization: Bearer search-soaewu2ye6uc45dr8mcd54v8' \
-d '{
  "filters": {
    "type": "curation"
  }
}'

The following fields are returned for each result:

  • query: Query that the Adaptive Relevance refers to
  • type: Adaptive Relevance Suggestion type (curation for now)
  • status: One of pending, applied, automated, rejected, disabled.
  • updated_at: Timestamp of the last update made to the suggestion.
  • created_at: Timestamp of the suggestion creation date and time.
  • promoted: Array of document identifiers that are being promoted by the suggestion.
  • operation: Identifies the operation to be done to the query curation. One of create, update or delete.
  • curation_id: (Optional) Curation identifier that the operation will be applied to.
  • override_curation_id: (Optional) Curation identifier for an status pending ,create operation that identifies the curation that will be overriden with the suggestion.

Adaptive Relevance Settings APIedit

Adaptive Relevance Settings are only available for Platinum Licenses.

Adaptive Relevance Settings are a set of engine-level configurations to determine how Adaptive Relevance Suggestions are used. Use this API to retrieve and manage Adaptive Relevance Settings.

  • Get Adaptive Relevance Settings for an Engine
  • Update Adaptive Relevance Settings for an Engine

Get Adaptive Relevance Settings for an Engineedit

Return the Adaptive Relevance Settings for an engine.

GET /api/as/v1/engines/{ENGINE_NAME}/adaptive_relevance/settings

Usage examples

Retrieve Adaptive Relevance Settings for an engine:

curl -X GET 'https://[instance id].ent-search.[region].[provider].cloud.es.io/api/as/v1/engines/national-parks-demo/adaptive_relevance/settings' \
-H 'Authorization: Bearer private-xxxxxxxxxxxxxxxxxxxxxxxx'

Response:

{
  "curation": {
    "enabled": true,
    "mode": "manual",
    "timeframe": 7,
    "max_size": 3,
    "min_doc_position": 3,
    "min_clicks": 20,
    "schedule_frequency": 1,
    "schedule_unit": "day"
  }
}

Update Adaptive Relevance Settingsedit

Update the Adaptive Relevance Settings for an engine.

PUT /api/as/v1/engines/{ENGINE_NAME}/adaptive_relevance/settings

Request body

curations

The Adaptive Relevance Settings for the curation type:

  • enabled: Boolean to determine if the Adaptive Relevance Suggestions is turned on for the engine and curation type.
  • mode: Must be manual or automatic. Curations are automatically applied if automatic. manual by default.
  • timeframe: Integer to determine the number of days worth of logs to use for the Adaptive Relevance Suggestions analysis. Defaults to 7.
  • max_size: Maximum number of promoted documents. Defaults to 3.
  • min_doc_position: consider only curation with at least a doc at the position. Defaults to 3.
  • min_clicks: Minimum number of clicks to be considered as significant. Defaults to 20.
  • schedule_frequency: Integer value used with schedule_unit to determine how frequently to run Adaptive Relevance Suggestions analysis. Defaults to 1.
  • schedule_unit: Value used with schedule_frequency to determine how frequently to run Adaptive Relevance Suggestions analysis. Must be one of second, minute, hour, day, week, month. Defaults to day.

All fields are optional.

Usage examples

Turn off Adaptive Relevance Suggestions for an engine:

curl -X PUT 'https://[instance id].ent-search.[region].[provider].cloud.es.io/api/as/v1/engines/national-parks-demo/adaptive_relevance/settings' \
-H 'Authorization: Bearer private-xxxxxxxxxxxxxxxxxxxxxxxx' \
-d '{ "enabled": false }'

Response:

{
  "curation": {
    "enabled": false,
    "mode": "manual",
    "timeframe": 7,
    "max_size": 3,
    "min_doc_position": 3,
    "min_clicks": 20,
    "schedule_frequency": 1,
    "schedule_unit": "day"
  }
}

Turn on automatic Adaptive Relevance Suggestions for an engine:

curl -X PUT 'https://[instance id].ent-search.[region].[provider].cloud.es.io/api/as/v1/engines/national-parks-demo/adaptive_relevance/settings' \
-H 'Authorization: Bearer private-xxxxxxxxxxxxxxxxxxxxxxxx' \
-d '{ "mode": "automatic" }'

Response:

{
  "curation": {
    "enabled": false,
    "mode": "automatic",
    "timeframe": 7,
    "max_size": 3,
    "min_doc_position": 3,
    "min_clicks": 20,
    "schedule_frequency": 1,
    "schedule_unit": "day"
  }
}

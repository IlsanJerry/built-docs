Explore Kibana using sample dataedit

Ready to get some hands-on experience with Kibana? In this tutorial, you’ll work with Kibana sample data and learn to:

  • Explore the sample data using Discover
  • View and analyze the data on a dashboard
  • Filter and query the dashboard data

If security is enabled, you must have read, write, and manage privileges on the kibana_sample_data_* indices. For more information, refer to Security privileges.

Add the sample dataedit

Add the Sample flight data.

  1. On the home page, click Load a data set and a Kibana dashboard.
  2. On the Sample flight data card, click Add data.

Explore the dataedit

Explore the documents in the index that match the selected index pattern. The index pattern tells Kibana which Elasticsearch index you want to explore.

  1. Open the menu, then go to Discover.

  2. Make sure kibana_sample_data_flights is the current index pattern. You might need to click New in the Kibana toolbar to refresh the data.

    You’ll see a histogram that shows the distribution of documents over time. A table lists the fields for each document that matches the index. By default, all fields are shown.

  3. Hover over the list of Available fields, then click Add next to each field you want explore in the table.

View and analyze the dataedit

A dashboard is a collection of panels that provide you with an overview of your data that you can use to analyze your data. Panels contain everything you need, including visualizations, interactive controls, Markdown, and more.

To open the Global Flight dashboard, open the menu, then go to Dashboard.

Change the panel dataedit

To gain insights into your data, change the appearance and behavior of the panels. For example, edit the metric panel to find the airline that has the lowest average fares.

  1. In the Kibana toolbar, click Edit.
  2. In the Average Ticket Price metric panel, open the panel menu, then select Edit visualization.

  3. To change the data on the panel, use an Elasticsearch bucket aggregation, which sorts the documents that match your search criteria into different categories or buckets.

    1. In the Buckets pane, select Add > Split group.
    2. From the Aggregation dropdown, select Terms.
    3. From the Field dropdown, select Carrier.

    4. Set Descending to 4, then click Update.

      The average ticket price for all four airlines appear in the visualization builder.

  4. To save your changes, click Save and return in the Kibana toolbar.

  5. To save the dashboard, click Save in the Kibana toolbar.

Filter and query the dataedit

To focus in on the data you want to explore, use filters and queries. For more information, refer to Query and filter context.

To filter the data:

  1. In the Controls visualization, select an Origin City and Destination City, then click Apply changes.

    The OriginCityName and the DestCityName fields filter the data in the panels.

    For example, the following dashboard shows the data for flights from London to Milan.

  2. To manually add a filter, click Add filter, then specify the data you want to view.
  3. When you are finished experimenting, remove all filters.

To query the data:

  1. To view all flights out of Rome, enter the following in the KQL query bar, then click Update:

    OriginCityName: Rome

  2. For a more complex query with AND and OR, enter:

    OriginCityName:Rome AND (Carrier:JetBeats OR Carrier:"Kibana Airlines")

    The dashboard panels update to display the flights out of Rome on JetBeats and Kibana Airlines.

  3. When you are finished exploring, remove the query by clearing the contents in the KQL query bar, then click Update.

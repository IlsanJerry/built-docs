Breaking changes in 8.0edit

Here are the breaking changes for 8.0.

Changes in Logstash Coreedit

Field Reference parser removededit

The Field Reference parser by default was STRICT in version 7. This configuration is removed permanently. Logstash keeps the behaviour as if the default setting, which interprets references to fields in your pipelines and plugins, and rejects inputs that are either ambiguous or illegal.

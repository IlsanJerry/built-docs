Upgrading Logstashedit

Before upgrading Logstash:

  • Consult the breaking changes docs.
  • Read the Release Notes.
  • Test upgrades in a development environment before upgrading your production cluster.

While upgrading Logstash:

  • If you use X-Pack monitoring, you must re-use the data directory when you upgrade Logstash. Otherwise, the Logstash node is assigned a new persistent UUID and becomes a new node in the monitoring data.

If you’re upgrading other products in the stack, also read the Elastic Stack Installation and Upgrade Guide. Want an upgrade list that’s tailored to your stack? Try out our Interactive Upgrade Guide.

Add anomaly detection jobs to calendar API

Adds an anomaly detection job to a calendar.

Request

PUT _ml/calendars/<calendar_id>/jobs/<job_id>

Prerequisites

  • You must have manage_ml, or manage cluster privileges to use this API. For more information, see Security privileges.

Path parameters

<calendar_id>
(Required, string) Identifier for the calendar.
<job_id>
(Required, string) An identifier for the anomaly detection jobs. It can be a job identifier, a group name, or a comma-separated list of jobs or groups.

Examples

The following example associates the planned-outages calendar with the total-requests job:

PUT _ml/calendars/planned-outages/jobs/total-requests

The API returns the following results:

{
  "calendar_id": "planned-outages",
  "job_ids": [
    "total-requests"
  ]
}

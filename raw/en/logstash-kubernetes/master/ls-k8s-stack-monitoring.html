Set up stack monitoringedit

This documentation is still in development and may be changed or removed in a future release.

In order to prepare your environment to be production ready, you’ll need to set up secure communication between each of your Elastic resources.

An important step to making your environment production ready is to configure stack monitoring. Monitoring logs and metrics data can be sent to an external resource, such as Elasticsearch Service or Elastic Cloud on Kubernetes, so that in the event that any components of your environment become unresponsive, your monitoring data is available.

ElasticsearchScriptTypeSettings

Scripts can be stored in indexes ('stored'), or uploaded in file bundles ('file'), or used directly in API requests ('inline'). This object controls whether scripting is enabled for the given type, together with other fine grained control parameters.

Properties

enabled (boolean)
If enabled (default: false in 5.x, true in 2.x) then scripts for all enabled scripting engines are allowed for this type of script. NOTES: (Corresponds to the parameter 'script.file|stored/indexed|inline')
sandbox_mode (boolean)
(2.x only - sandboxing is not required in 5.x because the supported script languages are safer). If enabled (default: true) and this script type is enabled, then all scripts are run inside a sandbox preventing insecure server-side operations. NOTES: (Corresponds to the parameter 'script.file|stored/indexed|inline')

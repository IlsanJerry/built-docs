Platform - Configuration - Security - Extra Certificates - CRUD

  • List extra certificates
  • Delete extra certificate
  • Read extra certificate
  • Update extra certificate

List extra certificates

List all of the extra certificates.

Request

GET /api/v1/platform/configuration/security/extra_certs

Responses

200

(PublicCertChainCollection) The collection of all available extra certificates public chains

To perform this operation, you must be authenticated by means of one of the following methods: apiKey, basicAuth.

Request example

curl -XGET https://{{hostname}}/api/v1/platform/configuration/security/extra_certs \
-H "Authorization: ApiKey $ECE_API_KEY"

Delete extra certificate

Delete an extra certificate.

Request

DELETE /api/v1/platform/configuration/security/extra_certs/{cert_id}

Path parameters

Name Type Required Description

cert_id

string

Y

The extra certificate Id. For example, star_my_domain_com.

Responses

200

(EmptyResponse) Extra certificate under the given id is deleted if exists

404

(BasicFailedReply) The extra certificate cannot be found. (code: cert.not_found)

Headers

x-cloud-error-codes (string; allowed values: [cert.not_found])
The error codes associated with the response

To perform this operation, you must be authenticated by means of one of the following methods: apiKey, basicAuth.

Request example

curl -XDELETE https://{{hostname}}/api/v1/platform/configuration/security/extra_certs/{cert_id} \
-H "Authorization: ApiKey $ECE_API_KEY"

Read extra certificate

Read an extra certificate.

Request

GET /api/v1/platform/configuration/security/extra_certs/{cert_id}

Path parameters

Name Type Required Description

cert_id

string

Y

The extra certificate Id. For example, star_my_domain_com.

Responses

200

(PublicCertChain) Extra certificate public chain if exists

404

(BasicFailedReply) The extra certificate cannot be found. (code: cert.not_found)

Headers

x-cloud-error-codes (string; allowed values: [cert.not_found])
The error codes associated with the response

To perform this operation, you must be authenticated by means of one of the following methods: apiKey, basicAuth.

Request example

curl -XGET https://{{hostname}}/api/v1/platform/configuration/security/extra_certs/{cert_id} \
-H "Authorization: ApiKey $ECE_API_KEY"

Update extra certificate

Add or update an extra certificate.

Request

PUT /api/v1/platform/configuration/security/extra_certs/{cert_id}

Path parameters

Name Type Required Description

cert_id

string

Y

The extra certificate Id. For example, star_my_domain_com.

Request body

(string) (required) Extra certificate bundle

Responses

200

(EmptyResponse) Updates extra certificate under given id

400

(BasicFailedReply) Certificate uses unknown cryptographic algorithm. (code: cert.unknown_cert_algorithm)

Headers

x-cloud-error-codes (string; allowed values: [cert.unknown_cert_algorithm])
The error codes associated with the response

To perform this operation, you must be authenticated by means of one of the following methods: apiKey, basicAuth.

Request example

curl -XPUT https://{{hostname}}/api/v1/platform/configuration/security/extra_certs/{cert_id} \
-H "Authorization: ApiKey $ECE_API_KEY"

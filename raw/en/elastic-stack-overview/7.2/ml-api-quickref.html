API quick reference

All machine learning anomaly detection endpoints have the following base:

/_ml/

The main resources can be accessed with a variety of endpoints:

  • /anomaly_detectors/: Create and manage machine learning jobs
  • /calendars/: Create and manage calendars and scheduled events
  • /datafeeds/: Select data from Elasticsearch to be analyzed
  • /filters/: Create and manage filters for custom rules
  • /results/: Access the results of a machine learning job
  • /model_snapshots/: Manage model snapshots

For a full list, see Machine learning anomaly detection APIs.
